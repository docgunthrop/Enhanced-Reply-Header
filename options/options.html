    Syntax for Custom Date (user-defined)

    Category Key Value
    Day of the Month d 2 digits ("01" to "31")
    Weekday D text ("Friday")
    Month m numeric ("01" to "12")
    M text ("January")
    Year y last 2 digits ("22")
    Y full 4 digits ("2022")
    Time i minutes ("00" to "59")
    s seconds ("00" to "59")
    Hour Format H 24-hour format ("00" to "23")
    h 12-hour format ("01" to "12")
    AM/PM A uppercase ("AM" / "PM")
    a lowercase ("am" / "pm")
    Timezone z short ("PDT")
    Z long ("Pacific Daylight Time")
    t offset ("GMT-07:00")
    • Wrap all keys with "%" (e.g. %Y% for year)
    • Keys can be followed by an integer that denotes the length of the value (e.g. %D3% for Friday returns "Fri")
    • When there is no integer appended to the key, the value defaults to its full length
    • For keys d, m, and h: setting integer to "1" (example: %m1%) results in the numeric value without leading zeroes
    • Categories highlighted in light-blue are only allowed one option (e.g. for "Month", either %m% or %M% can be used, but not both)
